Architectural Note

The "Repeat" phase is critical. Unlike standard DAGs, the agent loop is recursive. Ensure your memory buffers are cleared to prevent context tokens from exploding.

python v3.11+ 
async def agent_reasoning_loop(prompt: str, max_iterations: int = 10):
    # Initialize logical state context
    state = AgentState(input=prompt)
    
    while state.is_active and state.step < max_iterations:
        try:
            # Step 1: Model Prediction
            prediction = await model.predict(state.context)
            
            if prediction.type == "tool_call":
                # Step 2: Tool Execution with Error Handling
                result = await executor.run(prediction.tool, prediction.args)
                state.update_memory(observation=result)
            else:
                # Final result achieved
                state.is_active = False
                return prediction.content

        except ExecutionError as e:
            # Self-Correction Protocol
            logger.warning(f"Step {state.step} failed: {e}")
            state.inject_feedback(error_message=str(e))
            
        state.step += 1
        
    raise TimeoutError("Agent failed to converge within limits.")

Optimization Tip

Implementing exponential backoff in the ExecutionError catch block significantly improves the resilience of loops targeting unstable third-party APIs.
