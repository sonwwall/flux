API 接口定义

Synchronize your local architectural state with the global ledger. This endpoint performs an atomic resolution of pending transactions and returns the verified block hash.

POST /v2/ledger/sync

info
Requirement: This endpoint requires a Bearer Token with ledger.write scope.

Request Parameters

Name Type Description
vault_id string Unique identifier for the target architectural vault.
transaction_batch array An array of transaction objects to be synchronized.
strict_mode boolean Defaults to true. Fails if any conflict is detected.

Response Codes

200 OK — Sync completed successfully.
400 Bad Request — Invalid batch structure.
403 Forbidden — Insufficient permissions.

Interactive Playground

REQUEST EXAMPLE
bash 
curl -X POST https://api.archledger.io/v2/ledger/sync \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "vault_id": "v_77218x",
    "transaction_batch": [
      { "op": "ADD", "val": 450.00 },
      { "op": "REF", "id": "tx_99" }
    ],
    "strict_mode": true
  }'
RESPONSE BODY
json
200 OK 
{
  "status": "success",
  "ledger_hash": "0x8823...f110",
  "sync_time": "14.2ms",
  "verified_at": "2024-05-12T10:22:01Z"
}
verified_user

Pro Tip

Always use strict_mode in production to ensure total consistency across distributed ledger nodes. This prevents partial state commits during network partitions.
