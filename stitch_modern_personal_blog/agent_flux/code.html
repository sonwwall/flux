Agent 循环

自主系统（Autonomous Systems）的核心在于其自我维持的反馈循环。Agent 循环定义了系统如何接收外部刺激、处理上下文并执行具有长期影响的决策。

Input

环境感知 (Perception)

流式观测数据与静态上下文注入。

arrow_downward
Process

策略决策 (Reasoning)

通过 LLM 进行链式思考与逻辑推导。

执行动作 (Execution)

API 调用、系统变更或自然语言响应。

Feedback

核心原则：原子性循环

Agent 循环必须是原子的（Atomic）。这意味着每个循环周期应该具备独立的观测边界和明确的退出条件。避免在单个循环内处理过于复杂的子任务，建议将大型复杂逻辑分解为多层级的嵌套循环。

01. 感知层 (Perception)

这是循环的起点。Agent 通过配置的 Connectors 实时拉取外部世界的快照。这不仅包括原始文本，还包括结构化 JSON、数据库状态以及历史对话摘要。

  • 多模态输入解析（Text, Image, Logs）
  • 自适应上下文窗口管理

02. 决策层 (Decision)

在感知之后，Agent 进入 ReAct（Reasoning and Acting）阶段。Obsidian Flux 利用专有的 Prompt 架构确保 Agent 在执行前生成详细的执行计划。

  • 逻辑链 (CoT) 验证
  • 工具使用权限校验

ledger_agent.py 
class LedgerAgent(BaseAgent):
    """自主分类账处理 Agent"""
    
    def __init__(self, config: AgentConfig):
        super().__init__(config)
        self.cycle_count = 0

    async def run_loop(self, environment):
        while not self.check_termination():
            # 1. Perception
            state = await environment.get_current_state()
            
            # 2. Decision via Obsidian Reasoning Engine
            action = await self.reason(state)
            
            # 3. Execution
            result = await self.execute(action)
            
            # 4. Log & Memory update
            await self.memory.commit(result)
            self.cycle_count += 1
            
            print(f"Cycle {self.cycle_count} completed.")
